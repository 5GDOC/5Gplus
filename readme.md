## 2、协作规则

### 2.1 标题命名：

统一采用 属性tag +“|”+标题内容 的方式，例如：

- 清单| XXXX
- 卡片| XXXX

### 2.2 组成部分

- 标题
- 摘要
- 正文
- changelog

### 2.3 github中各板块的使用指南

- 库：沉淀资料、素材等
- issue：在探索过程或沟通中触发的灵感、改进、困难、协作等等都可放入
- project：团队分工协作
- wiki：作为结业作业中资料的最终整合

### 2.4 changelog的撰写

- 小标题：changelog：小写不空格
- 时间：6+4位数字：例如190101 1800
- 内容：人名+动词+名词：表明谁做了什么
- 顺序：新增的更新日志永远浮在最顶上，旧的更新日志逐渐沉底

### 2.5 关于创建、克隆、提交、push、pull、合并等

一定先克隆后编辑提交，以下工具可参考：

- GitHub Desktop
- SourceTree
